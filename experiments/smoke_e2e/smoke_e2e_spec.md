# Smoke E2E: gpt-2 forward pass

Tiny end-to-end test for zombuul itself. Loads gpt-2 small, runs a forward pass, saves internals to `results.json`. Used by `/zombuul:smoke-test` to exercise the full flow (`run-experiment` → pod → SSH → finalize → PR).

This is a test fixture, not real research. Don't move or rename — `/zombuul:smoke-test` looks for it at this exact path.

## Setup

- **GPU**: cheapest available (gpt-2 needs ~500 MB VRAM; an RTX A2000 is fine).
- **Model**: `gpt2` (124M params) from HuggingFace.
- **Input**: `"The quick brown fox jumps over the lazy"`.
- **Deps**: `transformers`, `torch` (install with `uv pip install transformers torch` on the pod if missing).

## Steps

1. Write `scripts/smoke_e2e/forward.py`:
   - Imports `torch` and `transformers`.
   - Loads `gpt2` model + tokenizer to GPU.
   - Encodes the input, runs a forward pass.
   - Takes argmax of last-position logits, decodes the predicted next token.
   - Computes L2 norm of last-position logits, checks for NaNs.
   - Saves to `experiments/smoke_e2e/results.json` (overwrite each run):
     ```json
     {
       "input": "<input>",
       "predicted_token_id": <int>,
       "predicted_token_str": "<str>",
       "logits_norm": <float>,
       "no_nans": <bool>,
       "device": "<str>"
     }
     ```
2. Run on the pod via SSH.
3. Rsync `experiments/smoke_e2e/results.json` back.

## Verification (in the report)

- `results.json` exists.
- `no_nans` is `true`.
- `logits_norm > 0`.
- `device` starts with `cuda`.
- `predicted_token_id` is a positive int.

Don't hardcode the expected token — gpt-2 is not bit-identical across GPU types.

## Outputs

- `experiments/smoke_e2e/results.json` (~300 bytes, committed).
- `experiments/smoke_e2e/smoke_e2e_report.md` (auto-generated by `finalize-experiment`).
